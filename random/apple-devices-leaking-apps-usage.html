The article doesn't mention why - searching for more information, that process and service appear to be used to check certificates used to sign apps, something that could be done in more efficient and private way. In principle, this check makes sense for ensuring safety of run applications, but in combination with required notarization it's sort of double-edged sword that forces developers to pay and comply with Apple if they want to distribute usable macOS software and users to let their computers call "home" whenever they try to run any 3rd-party software.
