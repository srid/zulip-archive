{-# LANGUAGE TypeApplications #-}

module Inference where

import Control.Carrier.Reader
import Data.Functor.Identity

test1 :: Int
test1 = runIdentity . runReader (10 :: Int) $ do
  x <- ask :: ReaderC Int Identity Int
  return x

test2 :: Int
test2 = runIdentity . runReader (10 :: Int) $ do
  x <- ask @Int
  return x

test3 :: Int
test3 = runIdentity $ runReader (10 :: Int) ask

test4 :: Int
test4 = runIdentity $ runReader (10 :: Int) (ask >>= return)

TheMatten
2021-03-09 10:42:56

@Pedro Minicz

ask :: Has (Reader r) sig m => m r
  • because return type is m r, it's not getting bound to r in ReaderC r in any way, so GHC has no way to find out what r is. When you add @Int, you tell it to search for Has (Reader Int) sig m specifically, which it finds because m = ReaderC Int Identity _

Problem with fundeps is that they don't allow you to add multiple Reader-s to your stack, because that would break assumption about stack determining r - polysemy "cheats" by not enforcing the dependency when there're multiple options in scope, requiring you to use type applications in those cases
